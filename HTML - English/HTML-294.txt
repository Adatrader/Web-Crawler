<h2><span class="mw-headline" id="Terminology"><span id="TERM"></span> Terminology</span></h2>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#TERM" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#TERM</a>&#160;</small></th></tr></tbody></table>
<dl><dd>On talk pages and in the edit summaries of a page history, you will often see editors using terminology and abbreviations which are unique to Wikipedia. The terms most likely to be unfamiliar to a new user are;
<dl><dd><b>rv</b> or <b>revert</b>, usually in an edit summary, indicates that the page has been reverted to a previous version, often because of vandalism.</dd>
<dd><b>NPOV</b> means working towards a <a href="/wiki/Neutral_point_of_view" class="mw-disambig" title="Neutral point of view">Neutral point of view</a>, whilst its opposite, <b>POV</b>, is used to suggest that an edit was biased.</dd>
<dd>To <b>Wikify</b> means to add internal links and other formatting to an article which was mostly plain text.</dd>
<dd><b>dab</b> means a small touch, and can also be short for disambiguation, or improving a link so that it goes straight to the relevant article.</dd></dl></dd>
<dd>For a more comprehensive list see <a href="/wiki/Wikipedia:Glossary" title="Wikipedia:Glossary">Wikipedia:Glossary</a>.</dd></dl>
<h3><span id="What_is_the_difference_between_a_page_and_an_article.3F"></span><span class="mw-headline" id="What_is_the_difference_between_a_page_and_an_article?">What is the difference between a page and an article?</span></h3>
<dl><dd>The term "page" encompasses all the material on Wikipedia, including encyclopedia topics, talk pages, documentation, and <a href="/wiki/Wikipedia:Special_pages" class="mw-redirect" title="Wikipedia:Special pages">special pages</a> such as <a href="/wiki/Special:RecentChanges" title="Special:RecentChanges">Recent Changes</a>. "Article" is a narrower term referring to a page containing an encyclopedia entry. Thus, all articles are pages, but not all pages are articles. See <a href="/wiki/Wikipedia:What_is_an_article" class="mw-redirect" title="Wikipedia:What is an article">Wikipedia:What is an article</a> for more.</dd></dl>

<h3><span id="What_is_a_minor_edit.3F_When_should_I_use_it.3F"></span><span class="mw-headline" id="What_is_a_minor_edit?_When_should_I_use_it?"><span id="MINOR"></span>What is a minor edit? When should I use it?</span></h3>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#MINOR" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#MINOR</a>&#160;</small></th></tr></tbody></table>
<dl><dd>When editing a page, a <a href="/wiki/Wikipedia:How_to_log_in" class="mw-redirect" title="Wikipedia:How to log in">logged-in</a> user has the option to flag an edit as "minor." Use of this flag is largely a matter of personal taste. A general rule of thumb is that an edit that corrects spelling or formatting, performs minor rearrangements of text, or tweaks only a few words, should generally be flagged as a "minor edit". A major edit, in contrast, generally performs a change that close watchers of the page are likely to want to review. Of course, if an edit performs a major semantic revision, but is limited to only a few words (for instance changing "freedom fighter" to "terrorist" or <i>vice versa</i> (see <a href="/wiki/Wikipedia:NPOV" class="mw-redirect" title="Wikipedia:NPOV">Wikipedia:NPOV</a>)), then the edit should <i>not</i> be flagged as minor.</dd></dl>
<dl><dd>This feature is important because users can choose to <i>hide</i> minor edits in their view of the <a href="/wiki/Special:RecentChanges" title="Special:RecentChanges">Recent Changes</a> page, to keep the volume of edits down to a manageable level.</dd></dl>
<dl><dd>Only <a href="/wiki/Wikipedia:How_to_log_in" class="mw-redirect" title="Wikipedia:How to log in">logged-in</a> users are allowed to mark an edit as minor.  The reason is that anonymous vandalism edits, if allowed to be marked as minor, could remain <i>hidden</i>, and therefore unnoticed, for longer than desired.  This limitation adds another reason to <a href="/wiki/Special:UserLogin" title="Special:UserLogin">create an account</a> and log in.</dd></dl>
<dl><dd>See also: <a href="/wiki/Wikipedia:Minor_edit" class="mw-redirect" title="Wikipedia:Minor edit">Wikipedia:Minor edit</a>.</dd></dl>

<h3><span id="Should_I_use_American_English_or_British_English.3F"></span><span class="mw-headline" id="Should_I_use_American_English_or_British_English?"><span id="ENGLISH"></span>Should I use <a href="/wiki/American_English" title="American English">American English</a> or <a href="/wiki/British_English" title="British English">British English</a>?</span></h3>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#ENGLISH" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#ENGLISH</a>&#160;</small></th></tr></tbody></table>
<dl><dd>People contribute to the <a href="/wiki/English_language" title="English language">English language</a> Wikipedia in every possible variety and dialect of formal written English. The English language Wikipedia particularly welcomes contributions from editors whose first language is not English. Still, it is generally good form to keep usage consistent within a given article. The official policy is to use British (AKA "<a href="/wiki/American_and_British_English_differences" class="mw-redirect" title="American and British English differences">Commonwealth</a>") spelling when writing about British (or <a href="/wiki/Commonwealth_of_Nations" title="Commonwealth of Nations">Commonwealth</a>) topics, and <a href="/wiki/American_English" title="American English">American</a> for topics relating to the <a href="/wiki/United_States" title="United States">United States</a>. General topics can use any one of the variants, but should generally strive to be consistent within an article. See Wikipedia's <a href="/wiki/Wikipedia:MoS#National_varieties_of_English" class="mw-redirect" title="Wikipedia:MoS">Manual of Style</a> for a more detailed explanation.</dd></dl>
<dl><dd>Use of one English variation in article titles can cause a <i>Search</i> in another variant to fail. In this case, it is recommended that you create a new article using the alternative spelling which <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirects</a> to the main article.</dd></dl>
<h3><span id="How_do_I_check_spelling_on_a_page.3F"></span><span class="mw-headline" id="How_do_I_check_spelling_on_a_page?"><span id="SPELL"></span>How do I check spelling on a page?</span></h3>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#SPELL" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#SPELL</a>&#160;</small></th></tr></tbody></table>
<dl><dd>A spelling checker has been requested for Wikipedia, but has not been implemented yet, except as a <a rel="nofollow" class="external text" href="https://code.google.com/p/wikimediaspellchecker/">third-party extension</a>. The <a href="/wiki/Firefox" title="Firefox">Firefox</a> 2.0 web browser automatically checks spelling in forms such as Wikipedia editing forms. When editing a larger article, it may be more convenient to paste the text into your favorite <a href="/wiki/Text_editor" title="Text editor">text editor</a> or <a href="/wiki/Word_processor" title="Word processor">word processor</a> first, to edit and check the spelling there, and then paste your corrected text back into Wikipedia editor to complete your contribution. You can also use an online spelling checker such as <a rel="nofollow" class="external text" href="http://www.spellonline.com">Spellonline</a>.</dd>
<dd>There is a <a href="/wiki/Wikipedia:List_of_common_misspellings" class="mw-redirect" title="Wikipedia:List of common misspellings">list of common misspellings</a>, which you can use to check if a listed misspelling is on any page in the database, although this process fails to identify any misspelled words not on that list. <a rel="nofollow" class="external text" href="http://www.google.com">Google</a> checks the spelling of words submitted to its search. Type a misspelled word into Google's search input field and Google places at the top of its search results a correctly spelled suggestion for what you might have meant to write.  For those who have the Google toolbar, it has a handy internet spelling checker.</dd></dl>

<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#CRASH" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#CRASH</a>&#160;</small></th></tr></tbody></table>
<dl><dd>In case of a crash you'll lose your edit. To some extent, you can guard against this by editing in a text editor, for major work (but note that with regard to a system crash this does not help, unless you save frequently to disk).</dd></dl>
<dl><dd>When you get a time-out when you try to save, you might or might not lose your edit depending on your <a href="/wiki/Web_browser" title="Web browser">web browser</a>. <h3><span id="Why_was_the_article_I_created_deleted.3F"></span><span class="mw-headline" id="Why_was_the_article_I_created_deleted?"><span id="DELETE"></span>Why was the article I created deleted?</span></h3>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#DELETE" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#DELETE</a>&#160;</small></th></tr></tbody></table>
<dl><dd>New articles are deleted for not following Wikipedia <a href="/wiki/Wikipedia:Policies_and_guidelines" title="Wikipedia:Policies and guidelines">policies and guidelines</a>. If your article was deleted, future contributions from you are still welcome.</dd>
<dd>The reasons that may lead to the quick deletion of an article are:
<ol><li>A very short page with little or no definition or context (e.g. "He is a funny man that has created Factory and the Hacienda. And, by the way, his wife is great.").</li>
<li>No meaningful content or history (e.g. "sdhgdf"). See <a href="/wiki/Wikipedia:Patent_nonsense" title="Wikipedia:Patent nonsense">patent nonsense</a>.</li>
<li>A test page (e.g. "Can I really create a page here?").</li>
<li>Pure vandalism (see <a href="/wiki/Wikipedia:Dealing_with_vandalism" class="mw-redirect" title="Wikipedia:Dealing with vandalism">dealing with vandalism</a>). Note that if you're not being malicious, then your article probably didn't fall under this category.</li>
<li>Reposted content that was deleted according to Wikipedia's <a href="/wiki/Wikipedia:Deletion_policy" title="Wikipedia:Deletion policy">deletion policy</a> unless undeleted according to <a href="/wiki/Wikipedia:Undeletion_policy" class="mw-redirect" title="Wikipedia:Undeletion policy">undeletion policy</a>.</li>
<li>A page created and edited solely by a banned user, after they were banned (see <a href="/wiki/Wikipedia:Bans_and_blocks" class="mw-redirect" title="Wikipedia:Bans and blocks">bans and blocks</a>).</li>
<li>An article which has already been moved via the <a href="https://meta.wikimedia.org/wiki/transwiki" class="extiw" title="m:transwiki">transwiki</a> system.</li>
<li>An article about a real person, group of people, band or club that does not assert the importance or significance of its subject.</li>
<li>An article that is a <b>blatant copyright infringement</b>  and contains no non-infringing revisions in its history.</li></ol></dd></dl>
<h3><span id="Why_was_the_edit_I_made_removed.3F"></span><span class="mw-headline" id="Why_was_the_edit_I_made_removed?"><span id="REMOVE"></span>Why was the edit I made removed?</span></h3>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#REMOVE" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#REMOVE</a>&#160;</small></th></tr></tbody></table>
<dl><dd>There are a variety of reasons (some <a href="/wiki/Wikipedia:Avoiding_common_mistakes" title="Wikipedia:Avoiding common mistakes">common reasons</a>).  The first thing you should do is look at the history page for the article you edited.  This will tell you who changed it, when they changed it, and hopefully a short reason why they changed it.  If it says something like <b>see talk</b>, then you should look at the talk page for the article. Also, you should look at your own talk page to see if you have a message there.  If you don't find a reason that is satisfactory, politely ask in the article's talk page about your proposed change, and maybe you will get suggestions about changes that you can make so that your change will go in, or you may get reasons why your change should not happen.</dd></dl>

See <a href="/wiki/Wikipedia:Translation" title="Wikipedia:Translation">Wikipedia:Translation</a> for further information.</dd></dl>
<h3><span id="What_about_using_machine_translation.3F"></span><span class="mw-headline" id="What_about_using_machine_translation?"><span id="MTRANS"></span>What about using machine translation?</span></h3>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#MTRANS" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#MTRANS</a>&#160;</small></th></tr></tbody></table>
<dl><dd><a href="/wiki/Machine_translation" title="Machine translation">Machine translation</a> is useful for obtaining the general idea of a text in an unfamiliar language, but it produces poor translations and should not be used on its own. If you want to use machine translation as a translation <i>aid</i> and intend to edit the result, please go ahead if you think it would be helpful. However, please do not paste a machine translation directly into an article.</dd></dl>
<h3><span id="How_can_I_tell_if_an_article_exists_in_another_language_Wikipedia.3F"></span><span class="mw-headline" id="How_can_I_tell_if_an_article_exists_in_another_language_Wikipedia?"><span id="OTHERLANG"></span>How can I tell if an article exists in another language Wikipedia?</span></h3>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#OTHERLANG" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#OTHERLANG</a>&#160;</small></th></tr></tbody></table>
<dl><dd>We try to build links between different language pages  that's one way of seeing if an article exists elsewhere. If you don't see the language links at the left of a page, go looking for the corresponding article(s) on foreign Wikipedias. If you find them, make a link both ways; if not, you can translate. Bear in mind that article may not be in one-to-one correspondence between Wikipedias. See <a href="/wiki/Wikipedia:Interlanguage_links" class="mw-redirect" title="Wikipedia:Interlanguage links">Wikipedia:Interlanguage links</a> and <a href="/wiki/Wikipedia:Multilingual_coordination" title="Wikipedia:Multilingual coordination">Wikipedia:Multilingual coordination</a> for more information.</dd></dl>
<h3><span id="Is_it_OK_to_link_to_other_sites.2C_as_long_as_the_material_is_not_copied_onto_Wikipedia.3F"></span><span class="mw-headline" id="Is_it_OK_to_link_to_other_sites,_as_long_as_the_material_is_not_copied_onto_Wikipedia?"><span id="OTHER"></span>Is it OK to link to other sites, as long as the material is not copied onto Wikipedia?</span></h3>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#OTHER" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#OTHER</a>&#160;</small></th></tr></tbody></table>
<dl><dd>External links are certainly allowed. Properly used, they increase the usability of Wikipedia. Keep in mind, however, that Wikipedia is not a <a href="/wiki/Web_directory" title="Web directory">web directory</a>; external links should support the content of the article, not replace it.  An article should be more than a container for external links, and the content should not require the reader to leave the site to understand the subject.</dd></dl>
<dl><dd>Please do not place advertising links in Wikipedia.  Commercial sites are obvious, but this prohibition <i>usually</i> includes links to fansites and discussion forums as well unless the site is a notable one in the field.  As a general rule of thumb: if you wish to place the link in Wikipedia in order to drive traffic to a site, it probably doesn't belong here.</dd></dl>
<dl><dd>The current convention is to place external links in a separate "External links" section at the bottom of the article.  Sites used as references for the article should be listed under a "References" section, or sometimes placed within the article as a footnote. See <a href="/wiki/Wikipedia:How_to_edit_a_page" class="mw-redirect" title="Wikipedia:How to edit a page">Wikipedia:How to edit a page</a> for different ways to create external links.</dd></dl>
<dl><dd>See the <a href="/wiki/Wikipedia:External_links" title="Wikipedia:External links">relevant guideline</a> for more examples on when external links are appropriate.</dd></dl>

<h2><span class="mw-headline" id="Copyrights">Copyrights</span></h2>
<h3><span id="I_have.2C_or_can_get.2C_special_permission_to_copy_an_image_or_article_to_Wikipedia._Is_it_OK_to_do_that.3F"></span><span class="mw-headline" id="I_have,_or_can_get,_special_permission_to_copy_an_image_or_article_to_Wikipedia._Is_it_OK_to_do_that?"><span id="IMAGE"></span>I have, or can get, special permission to copy an image or article to Wikipedia. Is it OK to do that?</span></h3>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#IMAGE" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#IMAGE</a>&#160;</small></th></tr></tbody></table>
<dl><dd>The text of Wikipedia is covered by the <a href="/wiki/Wikipedia:Text_of_Creative_Commons_Attribution-ShareAlike_3.0_Unported_License" title="Wikipedia:Text of Creative Commons Attribution-ShareAlike 3.0 Unported License">Creative Commons Attribution/Share-Alike License</a> (CC-BY-SA) and, in most cases, the <a href="/wiki/Wikipedia:Text_of_the_GNU_Free_Documentation_License" title="Wikipedia:Text of the GNU Free Documentation License">GNU Free Documentation License</a> (GFDL) (unversioned, with no invariant sections, front-cover texts, or back-cover texts). Images are covered by various free licenses (see the <a href="https://foundation.wikimedia.org/wiki/Resolution:Licensing_policy" class="extiw" title="wmf:Resolution:Licensing policy">Wikimedia Foundation Licensing Policy</a>). Unless the article you want to copy to Wikipedia is covered by CC-BY-SA, a CC-BY-SA-compatible license or is in the public domain, it cannot be used on Wikipedia. Images, too, must be public domain or licensed under one of allowed licenses at <a href="/wiki/Wikipedia:Image_copyright_tags" class="mw-redirect" title="Wikipedia:Image copyright tags">Wikipedia:Image copyright tags</a>. You have to ask the copyright holder of the material to license it so it can be used. See <a href="/wiki/Wikipedia:Requesting_copyright_permission" title="Wikipedia:Requesting copyright permission">Wikipedia:Requesting copyright permission</a>. Under very limited conditions we can use copyrighted material even if it is not licensed compatibly; see <a href="/wiki/Wikipedia:Non-free_content" title="Wikipedia:Non-free content">Wikipedia:Non-free content</a> for how and when.</dd></dl>
<h3><span id="I_have_an_out-of-copyright_image_.28or_text.29_that_is_reproduced_in_an_in-copyright_book._Can_I_scan_.2F_type_it_into_Wikipedia.3F"></span><span class="mw-headline" id="I_have_an_out-of-copyright_image_(or_text)_that_is_reproduced_in_an_in-copyright_book._Can_I_scan_/_type_it_into_Wikipedia?">I have an out-of-copyright image (or text) that is reproduced in an in-copyright book. Can I scan / type it into Wikipedia?</span></h3>
<dl><dd>Providing they haven't altered the image then they can't claim a copyright on it. If it was in the public domain before they used it, it's still in the public domain afterward.</dd></dl>
<h3><span id="Does_using_a_GIF_image_in_Wikipedia_violate_its_patent.3F"></span><span class="mw-headline" id="Does_using_a_GIF_image_in_Wikipedia_violate_its_patent?">Does using a <a href="/wiki/GIF" title="GIF">GIF</a> image in Wikipedia violate its patent?</span></h3>
<dl><dd>No. The patent of the <a href="/wiki/LZW" class="mw-redirect" title="LZW">LZW</a> compression algorithm used with the GIF format has expired.</dd></dl>

<h3><span id="Can_I_really_change_whatever_I_want_on_Wikipedia.3F"></span><span class="mw-headline" id="Can_I_really_change_whatever_I_want_on_Wikipedia?"><span id="ALL"></span>Can I really change whatever I want on Wikipedia?</span></h3>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#ALL" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#ALL</a>&#160;</small></th></tr></tbody></table>
<dl><dd>Yes, you can on almost any page. There are some pages on Wikipedia that are <a href="/wiki/Wikipedia:Protected_page" class="mw-redirect" title="Wikipedia:Protected page">protected</a>, so that only <a href="/wiki/Wikipedia:Administrators" title="Wikipedia:Administrators">administrators</a> can modify them. This includes pages like the <a href="/wiki/Main_Page" title="Main Page">Main Page</a>, which are permanently protected, or normal articles which are temporarily protected during the resolution of an <a href="/wiki/Wikipedia:Edit_war" class="mw-redirect" title="Wikipedia:Edit war">edit war</a>. Some pages are also <a href="/wiki/Wikipedia:Semi-protection" class="mw-redirect" title="Wikipedia:Semi-protection">semi-protected</a>, so that anonymous and new users cannot edit them, to reduce vandalism. The vast majority of pages on Wikipedia are editable by anyone, at any time, and any changes made will be reflected instantly.</dd></dl>
<h3><span id="How_do_I_get_a_count_of_my_edits.3F"></span><span class="mw-headline" id="How_do_I_get_a_count_of_my_edits?"><span id="COUNT"></span>How do I get a count of my edits?</span></h3>
<table class="shortcutbox shortcutbox-compact noprint" style="clear:right; float:right; border:1px solid #aaa; background:#fff; margin: 0.1em 0.1em 0.1em 0.1em; padding:1px; text-align:center;"><tbody><tr><th><small>&#160;<a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>: &#160;&#160;<a href="/wiki/Wikipedia:COFAQ#COUNT" class="mw-redirect" title="Wikipedia:COFAQ">WP:COFAQ#COUNT</a>&#160;</small></th></tr></tbody></table>
<dl><dd>There are a number of tools called <a href="/wiki/Wikipedia:Tools#Edit_counters" title="Wikipedia:Tools">edit counters</a> that can display the total count of your edits as well as the number of edits in various namespaces.</dd></dl>
<dl><dd>Your edit count, including deleted edits, is displayed when you enter <a href="/wiki/Special:Preferences" title="Special:Preferences">your preferences</a>.</dd></dl>
