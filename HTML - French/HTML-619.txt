<p><b>MediaWiki</b> est un <a href="/wiki/Moteur_de_wiki" title="Moteur de wiki">moteur de wiki</a> pour le <a href="/wiki/World_Wide_Web" title="World Wide Web">Web</a>. Il est utilisé par l’ensemble des projets de la <a href="/wiki/Wikimedia_Foundation" title="Wikimedia Foundation">Wikimedia Foundation</a>, des <a href="/wiki/Wiki" title="Wiki">wikis</a> hébergés chez <a href="/wiki/FANDOM" title="FANDOM">FANDOM</a>, ainsi que par de nombreux autres wikis. Conçu pour répondre aux besoins de <a href="/wiki/Wikip%C3%A9dia" title="Wikipédia">Wikipédia</a>, ce moteur est en 2008 également utilisé par des entreprises comme solution de <a href="/wiki/Gestion_des_connaissances" title="Gestion des connaissances">gestion des connaissances</a> et comme <a href="/wiki/Syst%C3%A8me_de_gestion_de_contenu" title="Système de gestion de contenu">système de gestion de contenu</a>. L’entreprise américaine <a href="/wiki/Novell" title="Novell">Novell</a> l’utilise notamment pour plusieurs de ses sites web qui véhiculent un trafic important. Des associations, comme  <a href="/wiki/Wikitravel" title="Wikitravel">Wikitravel</a>, <a href="/wiki/Mozilla_Foundation" title="Mozilla Foundation">Mozilla</a> ou <a href="/wiki/%C3%89kop%C3%A9dia" title="Ékopédia">Ékopedia</a>, l'ont aussi adopté.
</p><p>MediaWiki est écrit en <a href="/wiki/PHP" title="PHP">PHP</a> et peut aussi bien fonctionner avec le <a href="/wiki/Syst%C3%A8me_de_gestion_de_base_de_donn%C3%A9es" title="Système de gestion de base de données">système de gestion de base de données</a> <a href="/wiki/MySQL" title="MySQL">MySQL</a> que <a href="/wiki/PostgreSQL" title="PostgreSQL">PostgreSQL</a>. C'est un <a href="/wiki/Logiciel_libre" title="Logiciel libre">logiciel libre</a> distribué selon les termes de la <a href="/wiki/Licence_publique_g%C3%A9n%C3%A9rale_GNU" title="Licence publique générale GNU">GPL</a>.
</p><p>MediaWiki comporte de nombreuses fonctionnalités pour les sites à <a href="/wiki/Travail_collaboratif" title="Travail collaboratif">vocation collaborative</a>. Par exemple, la gestion des <a href="/wiki/Espace_de_noms" title="Espace de noms">espaces de noms</a>, ou l'utilisation de pages de discussions associées à chaque article.
</p>

<h2><span class="mw-headline" id="Histoire">Histoire</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=1" class="mw-editsection-visualeditor" title="Modifier la section : Histoire">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=1" title="Modifier la section : Histoire">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h2>
<div class="thumb tright"><div class="thumbinner" style="width:222px;"><a href="/wiki/Fichier:Usemod_logo.svg" class="image"><img alt="Logo de UseModWiki" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/e3/Usemod_logo.svg/220px-Usemod_logo.svg.png" decoding="async" width="220" height="220" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/e3/Usemod_logo.svg/330px-Usemod_logo.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/e3/Usemod_logo.svg/440px-Usemod_logo.svg.png 2x" data-file-width="100" data-file-height="100" /></a>  <div class="thumbcaption"><div class="magnify"><a href="/wiki/Fichier:Usemod_logo.svg" class="internal" title="Agrandir"></a></div>Logo de UseModWiki</div></div></div>
<p>Initialement, <a href="/wiki/Wikip%C3%A9dia" title="Wikipédia">Wikipédia</a> utilisait un <a href="/wiki/Moteur_de_wiki" title="Moteur de wiki">moteur de wiki</a> rudimentaire écrit en <a href="/wiki/Perl_(langage)" title="Perl (langage)">Perl</a>, appelé <a href="/wiki/UseModWiki" title="UseModWiki">UseModWiki</a>. Le <time class="nowrap" datetime="2002-01-25" data-sort-value="2002-01-25">25 janvier 2002</time>, MediaWiki, développé par <a href="/wiki/Magnus_Manske" title="Magnus Manske">Magnus Manske</a>, un étudiant <a href="/wiki/Allemagne" title="Allemagne">allemand</a> de l’<a href="/wiki/Universit%C3%A9_de_Cologne" title="Université de Cologne">université de Cologne</a>, devient le moteur de wiki de l'encyclopédie <a href="/wiki/Plate-forme_collaborative" title="Plate-forme collaborative">collaborative</a> pour laquelle il a été développé. MediaWiki a ainsi permis de disposer de plus de fonctionnalités et d’une infrastructure plus extensible (grâce à une <a href="/wiki/Base_de_donn%C3%A9es" title="Base de données">base de données</a> <a href="/wiki/MySQL" title="MySQL">MySQL</a>). Les performances du logiciel ont ensuite été améliorées par <a href="/wiki/Lee_Daniel_Crocker" title="Lee Daniel Crocker">Lee Daniel Crocker</a>, avant que Brion Vibber n'en devienne le développeur le plus actif et ne prenne le rôle de dirigeant des sorties logicielles<sup id="cite_ref-3" class="reference"><a href="#cite_note-3"><span class="cite_crochet">[</span>3<span class="cite_crochet">]</span></a></sup>.
</p><p>Depuis la sortie de la première version du script de Manske, plusieurs noms représentatifs de l’état du logiciel lui ont été donnés&#160;: «&#160;le script PHP&#160;», «&#160;phase II&#160;», «&#160;phase III&#160;», «&#160;le nouveau code source&#160;». Cependant, il n’était pourvu d’aucun nom de produit. Après l'annonce de la création de la <a href="/wiki/Wikimedia_Foundation" title="Wikimedia Foundation">Wikimedia Foundation</a> le <time class="nowrap" datetime="2003-06-20" data-sort-value="2003-06-20">20 juin 2003</time>, le <a href="/wiki/Communaut%C3%A9_wikip%C3%A9dienne" title="Communauté wikipédienne">wikipédien</a> Daniel Mayer lui donne le nom «&#160;MediaWiki&#160;», par jeu de mots sur le nom «&#160;Wikimedia&#160;», et ce nom est progressivement adopté. Pourtant, la similarité des noms MediaWiki et Wikimedia (qui lui-même est déjà semblable au nom Wikipédia) est à l'origine de fréquentes confusions.
</p><p>Le logo de MediaWiki a été créé par Erik Moeller à partir d’une photographie d’une fleur prise par <a href="/wiki/Florence_Devouard" title="Florence Devouard">Florence Devouard</a> (qui est par la suite devenue présidente de la <a href="/wiki/Wikimedia_Foundation" title="Wikimedia Foundation">Wikimedia Foundation</a>) et a initialement été soumis au concours international du nouveau logo pour Wikipédia qui s’est déroulé pendant l’été 2003. Le logo s'est placé en troisième position à l'issue de ce concours et a été choisi pour représenter MediaWiki plutôt que Wikipédia, tandis que le logo vainqueur a été adopté pour représenter Wikipédia, et le second vainqueur pour la Wikimedia Foundation. Les doubles crochets sur la photo autour du <a href="/wiki/Tournesol" title="Tournesol">tournesol</a> symbolisent le <a href="/wiki/Wikitexte" title="Wikitexte">wikicode</a>, c'est-à-dire la <a href="/wiki/Syntaxe" title="Syntaxe">syntaxe</a> utilisée par MediaWiki pour créer des <a href="/wiki/Hyperlien" title="Hyperlien">hyperliens</a> vers les autres pages du wiki.
</p>

<h2><span id="Fonctionnalit.C3.A9s"></span><span class="mw-headline" id="Fonctionnalités">Fonctionnalités</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=4" class="mw-editsection-visualeditor" title="Modifier la section : Fonctionnalités">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=4" title="Modifier la section : Fonctionnalités">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>MediaWiki est muni d’un ensemble de <a href="/wiki/Moteur_de_wiki#Fonctionnalités_évoluées" title="Moteur de wiki">fonctionnalités élaborées</a> et d’un système d’<a href="/wiki/Extension_(logiciel)" title="Extension (logiciel)">extensions</a> lui permettant d’intégrer des fonctions supplémentaires. En raison de la forte présence du plurilinguisme dans les projets <a href="/wiki/Wikimedia_Foundation" title="Wikimedia Foundation">Wikimedia</a>, la partie <a href="/wiki/Internationalisation" title="Internationalisation">internationalisation</a> a bénéficié d’une attention toute particulière de la part des développeurs. L’interface a été intégralement ou partiellement traduite dans <span class="nowrap">352 langues</span><sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span class="cite_crochet">[</span>7<span class="cite_crochet">]</span></a></sup> et peut être personnalisée par les administrateurs du site. <a href="/wiki/Wikip%C3%A9dia" title="Wikipédia">Wikipédia</a> étant l’un des sites web les plus visités au monde<sup id="cite_ref-alexa.com_8-0" class="reference"><a href="#cite_note-alexa.com-8"><span class="cite_crochet">[</span>8<span class="cite_crochet">]</span></a></sup><sup class="reference cite_virgule">,</sup><sup id="cite_ref-9" class="reference"><a href="#cite_note-9"><span class="cite_crochet">[</span>9<span class="cite_crochet">]</span></a></sup>, les développeurs ont apporté un soin tout particulier à l’<a href="/wiki/Extensibilit%C3%A9" class="mw-redirect" title="Extensibilité">extensibilité</a> grâce à un système multicouche de <a href="/wiki/Mise_en_cache" class="mw-redirect" title="Mise en cache">mise en cache</a> et de <a href="/w/index.php?title=Duplication_de_base_de_donn%C3%A9es&amp;action=edit&amp;redlink=1" class="new" title="Duplication de base de données (page inexistante)">duplication de base de données</a>. En 2008, Wikipédia et les autres projets Wikimedia sont toujours à l’origine d’une grande partie des critères de réalisation de MediaWiki.
</p>
<h3><span class="mw-headline" id="Liens_libres">Liens libres</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=5" class="mw-editsection-visualeditor" title="Modifier la section : Liens libres">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=5" title="Modifier la section : Liens libres">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Le principal atout de MediaWiki par rapport aux autres <a href="/wiki/Moteur_de_wiki" title="Moteur de wiki">moteurs de wiki</a> est l’usage de liens libres au lieu du <i><a href="/wiki/Camel_case" title="Camel case">camel case</a></i>. Là où habituellement les autres wikis ont besoin d’écrire «&#160;WorldWideWeb&#160;» en un seul mot pour produire un lien vers la page du même nom, MediaWiki n’impose aucune contrainte, il suffit de placer l’expression que l’on souhaite lier entre doubles crochets. Tous les espaces contenus dans l’expression sont ainsi préservés, comme pour [[World Wide Web]]. Néanmoins, certains caractères ne peuvent pas être utilisés, par exemple # ou |.
</p>
<h3><span class="mw-headline" id="Espaces_de_nom">Espaces de nom</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=6" class="mw-editsection-visualeditor" title="Modifier la section : Espaces de nom">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=6" title="Modifier la section : Espaces de nom">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>En plus des liens, MediaWiki est pourvu de nombreuses autres fonctionnalités pour structurer le contenu. L’une des plus anciennes fonctionnalités est le système d’<a href="/wiki/Espace_de_nom" class="mw-redirect" title="Espace de nom">espace de nom</a>. Pendant longtemps, la séparation des discussions autour de Wikipédia et de son contenu encyclopédique a constitué un réel problème&#160;; de même concernant les pages personnelles des wikipédiens. De façon simpliste, les espaces de nom peuvent se résumer à des préfixes utilisés dans le titre des pages (par exemple «&#160;Utilisateur:&#160;» ou bien «&#160;Discussion:&#160;»), ce qui permet à plusieurs pages d'exister sous le même nom (à l'espace de nom près), mais avec des finalités différentes selon le préfixe. Par exemple, la page intitulée «&#160;[[Terminator]]&#160;» peut décrire le <a href="/wiki/Terminator" title="Terminator">film de 1984</a> réalisé par <a href="/wiki/James_Cameron" title="James Cameron">James Cameron</a>, tandis que la page «&#160;[[Utilisateur:Terminator]]&#160;» peut être une page personnelle décrivant un utilisateur ayant choisi ce pseudonyme. Chaque page de Wikipédia est associée à une page de discussion qui a pour but de permettre aux wikipédiens de discuter du sujet de la page.
</p><p>Par comparaison avec les systèmes de fichiers, les espaces de nom peuvent être assimilés aux <a href="/wiki/R%C3%A9pertoire_(informatique)" title="Répertoire (informatique)">dossiers</a> qui permettent de séparer plusieurs fichiers de même nom. Bien qu’il soit possible de rajouter des espaces de nom, leur nombre dans un wiki est habituellement relativement faible.
</p>
<h3><span class="mw-headline" id="Sous-pages">Sous-pages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=7" class="mw-editsection-visualeditor" title="Modifier la section : Sous-pages">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=7" title="Modifier la section : Sous-pages">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>En plus des espaces de noms, les pages peuvent être structurées à l’aide de sous-pages. Une page ayant pour titre «&#160;A&#160;» peut posséder plusieurs sous-pages dont le titre est alors de la forme «&#160;A/x&#160;». La sous-page propose alors automatiquement un <a href="/wiki/Lien_de_retour" class="mw-redirect" title="Lien de retour">lien de retour</a> vers la page mère (de la forme «&#160;&lt; A&#160;»). Dans cette dernière, on peut accéder à l'une de ses sous-pages en omettant son titre&#160;; ainsi, le lien «&#160;[[/x]]&#160;» pointera vers la sous-page ayant pour titre «&#160;A/x&#160;». Pour un espace de nom donné, le système de sous-pages peut être activé ou désactivé. Par exemple, sur Wikipédia, il est désactivé dans l'espace encyclopédique et activé dans l'espace «&#160;Utilisateur:&#160;».
</p>
<h3><span id="Cat.C3.A9gories"></span><span class="mw-headline" id="Catégories">Catégories</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=8" class="mw-editsection-visualeditor" title="Modifier la section : Catégories">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=8" title="Modifier la section : Catégories">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>MediaWiki permet aux utilisateurs de créer des catégories. Elles fonctionnent de façon similaire aux <a href="/wiki/Tag_(informatique)" class="mw-redirect" title="Tag (informatique)">tags</a> présents dans de nombreuses applications web et forment une hiérarchie et une description. Dans certains wikis comme Wikipédia, des hiérarchies complexes se sont développées avec ce système sans pour autant qu’une quelconque planification n’ait été établie.
</p>
<h3><span class="mw-headline" id="Interface_modifiable">Interface modifiable</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=9" class="mw-editsection-visualeditor" title="Modifier la section : Interface modifiable">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=9" title="Modifier la section : Interface modifiable">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>L’interface de MediaWiki peut être intégralement changée directement sur le wiki lui-même par les utilisateurs qui en ont le droit (il s’agit habituellement des personnes qui sont appelées <a href="/wiki/Wikip%C3%A9dia:Administrateur" title="Wikipédia:Administrateur">administrateurs</a> ou <i>sysops</i>). Cela se fait grâce à un espace de nom particulier qui correspond au préfixe «&#160;MediaWiki:&#160;» où chaque page correspond à un message précis de l’interface. À l’origine, l’espace de nom «&#160;MediaWiki:&#160;» était utilisé pour créer des blocs de texte personnalisés pouvant être chargés dynamiquement dans les autres pages à l’aide d’une syntaxe spéciale. Ce contenu a été déplacé plus tard dans un espace de nom à part appelé «&#160;Modèle:&#160;».
</p>
<h3><span id="Mod.C3.A8les"></span><span class="mw-headline" id="Modèles">Modèles</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=10" class="mw-editsection-visualeditor" title="Modifier la section : Modèles">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=10" title="Modifier la section : Modèles">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Les modèles sont des morceaux de texte qui peuvent être chargés de façon dynamique ou statique dans n'importe quelle page. Pour cela il suffit d’appeler le modèle en écrivant son nom entre doubles <a href="/wiki/Accolade" title="Accolade">accolades</a> (il s’agit en quelque sorte d’un lien spécial). Les modèles possèdent leur propre espace de nom qui correspond au préfixe «&#160;Modèle:&#160;». Par exemple pour faire appel dynamiquement au modèle se trouvant à la page intitulée «&#160;Modèle:Annexe&#160;», il suffit d’écrire «&#160;{{annexe}}&#160;». Le contenu du modèle sera ainsi reproduit dans la page où il est appelé, en temps réel. Si des changements sont apportés au modèle par la suite, la page sera également mise à jour automatiquement.
</p><p>Le modèle peut également être appelé à l'aide du préfixe «&#160;subst:&#160;» (exemple&#160;: {{subst:annexe}}), ce qui aura pour effet de substituer l'appel au modèle par son contenu au moment où la modification est validée. Le résultat est équivalent à un <a href="/wiki/Copier-coller" title="Copier-coller">copier-coller</a> du contenu du modèle. Par conséquent, tout changement apporté au modèle n’aura aucun effet dans la page dans laquelle le modèle y a été substitué. La substitution limite la cohérence de l’utilisation des modèles, mais peut être pratique dans certains cas, et est <i>a priori</i> moins gourmande en ressources <a href="/wiki/Serveur_(informatique)" class="mw-redirect" title="Serveur (informatique)">serveur</a>.
</p><p>Il est aussi possible de créer des modèles d'un espace de nom autre que modèle. Exemple&#160;: Toto est administrateur. Il souhaite annoncer facilement à un utilisateur qu'il le bloque et peu d'utilisateurs ont l'occasion d'utiliser ce modèle&#160;: il peut le créer dans son espace utilisateur sous un nom du type [[Utilisateur:Toto/Blocage]]. Il pourra alors l'utiliser sous la forme {{Utilisateur:Toto/Blocage}} ou {{subst:Utilisateur:Toto/Blocage}}. Il n'est pas possible d'insérer des portions d'articles de l'espace encyclopédique ou de pages de discussions sous forme de modèle ou de substitution de modèle.
</p><p>Les modèles sont utilisés de nombreuses manières différentes, par exemple&#160;:
</p>
<ul><li>l’identification de problème avec un article de Wikipédia en plaçant un modèle dans l’article. Le modèle produit un bandeau indiquant le type de problème rencontré dans l’article, et si le modèle a été prévu pour, la page peut être automatiquement catégorisée de sorte que tous les articles sur lesquels ce modèle est apposé soient tous regroupés dans une même catégorie&#160;;</li>
<li>la création de tables ayant une syntaxe compliquée, et dont l'utilisation est rendue simple dans les pages par l'utilisation de paramètres.</li></ul>
<h3><span class="mw-headline" id="Scripts">Scripts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=11" class="mw-editsection-visualeditor" title="Modifier la section : Scripts">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=11" title="Modifier la section : Scripts">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Il est possible d'invoquer des scripts en langage <a href="/wiki/Lua" title="Lua">Lua</a>, via la commande {{#invoke|Script|argument1|...}}.
</p>
<h3><span id="Multim.C3.A9dia"></span><span class="mw-headline" id="Multimédia">Multimédia</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=12" class="mw-editsection-visualeditor" title="Modifier la section : Multimédia">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=12" title="Modifier la section : Multimédia">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Comme le laisse supposer le nom MediaWiki, l’un des points forts du logiciel est sa capacité à gérer une large variété de fichiers multimédias (ces derniers peuvent être envoyés sur le wiki directement grâce à l’interface), en particulier les fichiers d'images, mais aussi les sons (au format <a href="/wiki/Ogg" title="Ogg">Ogg</a>). Si le logiciel est bien configuré, il est alors relativement facile de produire des galeries d’images et des vignettes. Le logiciel gère également les <a href="/wiki/M%C3%A9tadonn%C3%A9e" title="Métadonnée">métadonnées</a> <a href="/wiki/Exchangeable_image_file_format" title="Exchangeable image file format">Exif</a>. La nécessité de faire tourner <a href="/wiki/Wikimedia_Commons" title="Wikimedia Commons">Wikimedia Commons</a> (l’une des plus grandes archives de média de <a href="/wiki/%C5%92uvre_libre" title="Œuvre libre">contenu libre</a>) avec MediaWiki a été source d’apport en fonctionnalités de ce type.
</p><p>En février 2008, <a href="/wiki/Florence_Devouard" title="Florence Devouard">Florence Devouard</a> annonce la prochaine apparition d'une forme de vidéo collaborative sur Wikipédia<sup id="cite_ref-10" class="reference"><a href="#cite_note-10"><span class="cite_crochet">[</span>10<span class="cite_crochet">]</span></a></sup>. Il s'agirait d'utiliser un <a href="/wiki/Logiciel_libre" title="Logiciel libre">logiciel libre</a> conçu par la société <a href="/wiki/Kaltura" title="Kaltura">Kaltura</a>.
</p>
<h3><span id="Tel-quel_.28WYSIWYG.29"></span><span class="mw-headline" id="Tel-quel_(WYSIWYG)">Tel-quel (WYSIWYG)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=13" class="mw-editsection-visualeditor" title="Modifier la section : Tel-quel (WYSIWYG)">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=13" title="Modifier la section : Tel-quel (WYSIWYG)">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Depuis 2015, MediaWiki est pourvu d’une interface <a href="/wiki/What_you_see_is_what_you_get" title="What you see is what you get">WYSIWYG</a>, nommée VisualEditor.
</p><p>→ <a href="https://en.wikipedia.org/wiki/VisualEditor" class="extiw" title="en:VisualEditor">voir l'article détaillé : VisualEditor (en anglais)</a>
</p>
<h3><span class="mw-headline" id="Sections">Sections</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=14" class="mw-editsection-visualeditor" title="Modifier la section : Sections">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=14" title="Modifier la section : Sections">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Il est possible avec MediaWiki de modifier une partie d’une page (une section), ce qui permet notamment de modifier plus facilement et de façon concurrente des pages longues. En effet, si un utilisateur tente de valider des changements sur une page alors qu'un autre utilisateur l'a modifiée entretemps, il se produira alors un «&#160;conflit d'édition&#160;», et l'utilisateur devra effectuer une manipulation pour intégrer ses changements à la nouvelle version. En revanche, si deux utilisateurs modifient en même temps deux sections différentes d'une même page, ce problème ne se posera pas.
</p>
<h3><span id="Contenu_.C3.A9labor.C3.A9"></span><span class="mw-headline" id="Contenu_élaboré">Contenu élaboré</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=15" class="mw-editsection-visualeditor" title="Modifier la section : Contenu élaboré">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=15" title="Modifier la section : Contenu élaboré">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>MediaWiki est également muni de fonctionnalités permettant de gérer du contenu élaboré grâce à une syntaxe spéciale. Par exemple, le logiciel permet de produire des formules mathématiques à l’aide de <a href="/wiki/LaTeX" title="LaTeX">LaTeX</a> et d’un analyseur syntaxique écrit en <a href="/wiki/Objective_Caml" class="mw-redirect" title="Objective Caml">OCaml</a>. D’autres fonctionnalités de la sorte existent sous forme d’extension, notamment les <a href="/wiki/Frise_chronologique" title="Frise chronologique">frises chronologiques</a>, le tracé de <a href="/wiki/Courbe" title="Courbe">courbes mathématiques</a>, les <a href="/wiki/Partition_(musique)" title="Partition (musique)">partitions de musique</a> ou encore les <a href="/wiki/%C3%89criture_hi%C3%A9roglyphique_%C3%A9gyptienne" title="Écriture hiéroglyphique égyptienne">hiéroglyphes</a> égyptiens.
</p>
<h3><span class="mw-headline" id="Interface_personnalisable">Interface personnalisable</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=16" class="mw-editsection-visualeditor" title="Modifier la section : Interface personnalisable">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=16" title="Modifier la section : Interface personnalisable">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Si l’option est activée, les utilisateurs peuvent aussi personnaliser leur <a href="/wiki/Feuilles_de_style_en_cascade" title="Feuilles de style en cascade">feuille de style</a> et configurer du code <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> du côté du client pour qu’il soit exécuté sur chaque page visitée. Par exemple, le système de navigation par <a href="/wiki/Pop-up" title="Pop-up">pop-up</a> de Lupin est un outil JavaScript qui montre des aperçus d’articles lorsque l’utilisateur passe la souris au-dessus d’un lien, et donnent également des liens vers les tâches de maintenance habituelles. <a href="/wiki/Utilisateur:Leag/wikEd" title="Utilisateur:Leag/wikEd">wikEd</a> en est un autre exemple, un éditeur de texte intégré à MediaWiki, qui ne fonctionne qu'avec les navigateurs web de la <a href="/wiki/Utilisateur:Leag/wikEd" title="Utilisateur:Leag/wikEd">famille Mozilla</a>, avec de nombreuses fonctionnalités qui apporte la coloration syntaxique et des fonctions de rechercher et de remplacement<sup id="cite_ref-11" class="reference"><a href="#cite_note-11"><span class="cite_crochet">[</span>11<span class="cite_crochet">]</span></a></sup>.
</p>
<h3><span id="Groupes_et_droits_d.27acc.C3.A8s"></span><span class="mw-headline" id="Groupes_et_droits_d'accès">Groupes et droits d'accès</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=17" class="mw-editsection-visualeditor" title="Modifier la section : Groupes et droits d&#039;accès">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=17" title="Modifier la section : Groupes et droits d&#039;accès">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Bien que MediaWiki possède de base un ensemble de fonctions liées à la restriction, l’accès et la définition de groupes d’utilisateurs, le contrôle de l’accès aux pages ne semble pas constituer un élément de haute priorité dans le processus de développement. Par exemple, il n’est pas possible de définir des permissions d’accès aux pages en fonction de l’espace de nom. D'autres moteurs de wiki, comme <a href="/wiki/TWiki" title="TWiki">TWiki</a> et <a href="/wiki/MoinMoin" title="MoinMoin">MoinMoin</a>, apportent plus de puissance en gérant des mécanismes de sécurité avancés comme des listes de contrôle d’accès.
</p>
<h3><span class="mw-headline" id="Extensions">Extensions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=18" class="mw-editsection-visualeditor" title="Modifier la section : Extensions">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=18" title="Modifier la section : Extensions">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Le code source de MediaWiki contient plusieurs «&#160;crochets&#160;» («&#160;<i><span class="lang-en" lang="en">hooks</span></i>&#160;» en anglais) auxquels peut être attaché du code supplémentaire. Cela permet aux programmeurs d’écrire des extensions sans changer la base ou sans les obliger à soumettre leur code à un contrôle de révision. La plupart du temps, installer une extension revient à ajouter une ligne de code informatique dans le fichier de configuration, bien que dans certains cas, des changements au niveau de la base de données soient en plus nécessaires.
</p><p>De nombreuses extensions disponibles sont de simples scripts permettant d’intégrer du contenu comme des fichiers <a href="/wiki/Adobe_Flash" title="Adobe Flash">Adobe Flash</a> ou des formulaires <a href="/wiki/Extensible_HyperText_Markup_Language" class="mw-redirect" title="Extensible HyperText Markup Language">HTML</a>. D’autres scripts ajoutent de nouveaux comportements compliqués à la syntaxe wiki. C’est le cas de <a href="/wiki/Semantic_MediaWiki" title="Semantic MediaWiki">Semantic MediaWiki</a> qui donne la possibilité d’ajouter des relations structurées et recherchables et des attributs aux pages wiki (voir&#160;: <a href="/wiki/Web_s%C3%A9mantique" title="Web sémantique">Web sémantique</a>). La Wikimedia Foundation détient un serveur sur lequel de nombreuses extensions sont hébergées.
</p>
<h2><span class="mw-headline" id="Performances">Performances</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=19" class="mw-editsection-visualeditor" title="Modifier la section : Performances">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=19" title="Modifier la section : Performances">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Étant donné que MediaWiki est utilisé pour faire tourner <a href="/wiki/Wikip%C3%A9dia" title="Wikipédia">Wikipédia</a>, qui est un des dix sites les plus visités au monde<sup id="cite_ref-alexa.com_8-1" class="reference"><a href="#cite_note-alexa.com-8"><span class="cite_crochet">[</span>8<span class="cite_crochet">]</span></a></sup>, les performances du moteur et son <a href="/wiki/Extensibilit%C3%A9" class="mw-redirect" title="Extensibilité">extensibilité</a> ont été optimisées de façon accrue. MediaWiki gère les caches <a href="/wiki/Squid_(logiciel)" title="Squid (logiciel)">Squid</a>, les duplications de bases de données avec <a href="/wiki/R%C3%A9partition_de_charge" title="Répartition de charge">répartition de charge</a>, la mise en cache du côté du client, le <i><a href="/wiki/Memcached" title="Memcached">memcached</a></i> ou la mise en cache basée sur les tables pour les traitements fréquemment consultés ou les résultats de demandes courantes, sous la forme d’un fichier de cache statique, avec des fonctionnalités réduites, la compression des vieilles versions, et une file d’attente des tâches pour les opérations de base de données.
</p><p>Il se peut que MediaWiki soit surdimensionné pour un usage de taille réduite, car son large ensemble de fonctionnalités et son interface ont un coût en matière de performance. D’un autre côté, le logiciel convient parfaitement pour les opérations des <a href="/wiki/Ferme_de_wiki" class="mw-redirect" title="Ferme de wiki">fermes wikis</a>, comme les projets Wikimedia et la famille de langue. Cependant, MediaWiki n’est pas pourvu de fonctionnalités pour gérer de telles installations.
</p>
<h2><span class="mw-headline" id="Limites">Limites</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=MediaWiki&amp;veaction=edit&amp;section=20" class="mw-editsection-visualeditor" title="Modifier la section : Limites">modifier</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=MediaWiki&amp;action=edit&amp;section=20" title="Modifier la section : Limites">modifier le code</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Plusieurs remarques peuvent être faites à propos de MediaWiki&#160;: sa documentation est majoritairement disponible sur Internet, mais elle n'est pas propriétaire (des dizaines d'auteurs ont participé à sa rédaction). L'installation et l'utilisation du logiciel MediaWiki ne sont pas très intuitives pour un utilisateur inexpérimenté (comparé à d'autres logiciels, comme <a href="/wiki/Microsoft_FrontPage" title="Microsoft FrontPage">FrontPage</a> par exemple)<sup class="need_ref_tag" style="padding-left:2px;"><a href="/wiki/Aide:R%C3%A9f%C3%A9rence_n%C3%A9cessaire" title="Aide:Référence nécessaire">[réf.&#160;nécessaire]</a></sup>.
</p><p>Il n'y a pas de définition formelle de la syntaxe de MediaWiki&#160;; la syntaxe valide est celle reconnue par le code PHP de l'analyseur syntaxique, et il se peut qu'une définition de la syntaxe sous la forme <a href="/wiki/Extended_Backus-Naur_Form" title="Extended Backus-Naur Form">EBNF</a> soit impossible. C'est entre autres pour cette raison que produire un éditeur WYSIWYG ou une adaptation complète de l'analyseur pour d'autres langages de programmation est une tâche difficile.
</p><p>De plus, il n'existe pas de solution satisfaisante pour mettre en place une <a href="/wiki/Access_Control_List" title="Access Control List">liste de contrôle d'accès</a>.
</p><p>L'usage de deux types de liens (simples crochets pour des liens externes, doubles crochets pour des liens internes) est source de confusion et dispensable, comme l'a montré <a href="/wiki/Wikicr%C3%A9ole" title="Wikicréole">Wikicréole</a><sup class="need_ref_tag" style="padding-left:2px;"><a href="/wiki/Aide:R%C3%A9f%C3%A9rence_n%C3%A9cessaire" title="Aide:Référence nécessaire">[réf.&#160;nécessaire]</a></sup>.
</p>
